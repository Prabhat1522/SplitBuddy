# AI Collaboration Log: SplitBuddy

This document details the interaction with Antigravity AI during the development of SplitBuddy, highlighting prompt strategies, design collaborations, and debugging case studies.

---

## 1. Collaboration Overview
The SplitBuddy application was built through an interactive pair-programming partnership. 
- **AI Role**: Technical architect, database designer, and developer (drafting raw SQL transactions, greedy minimization algorithms, and React components).
- **User Role**: Product Manager and QA Engineer (specifying final schema constraints, choosing raw `mysql2` over Sequelize, defining multi-currency converting criteria, and inputting configurations).

---

## 2. Prompting Strategies
- **Detailed Requirement Scoping**: The initial architecture, database normalizations, and Gantt charts were drafted inside the `implementation_plan.md` artifact to obtain approval before writing code.
- **Incremental Commit Focus**: To avoid massive, unreviewable file dumps, the codebase was structured into 7 modular git-ready phases.

---

## 3. Case Studies: AI Code Corrections & Debugging Logs

### Case Study 1: MySQL 8.0 Reserved Keyword Conflict (`groups` table)
- **Initial Code Generated by AI**:
  ```sql
  CREATE TABLE groups (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL
  );
  ```
- **Error Observed**: During database setup queries, the server crashed with:
  `ER_PARSE_ERROR: You have an error in your SQL syntax... near 'groups'`
- **Correction**: In MySQL 8.0, `groups` is a reserved word. The AI corrected the schema and all Javascript SQL strings by wrapping the identifier in backticks:
  ```sql
  CREATE TABLE `groups` ( ... );
  ```

### Case Study 2: Multi-Statement Schema Seeding Error
- **Initial Code Generated by AI**:
  The database initializer attempted to run the entire `schema.sql` file (multiple statements) in a single query using the default pool connection.
- **Error Observed**: The server crashed with SQL parse exceptions on the second table statement.
- **Correction**: The AI resolved the issue by enabling `multipleStatements: true` in a temporary connection parameter map for database setups:
  ```javascript
  const setupConnection = await mysql.createConnection({
    ...
    multipleStatements: true
  });
  ```

### Case Study 3: Sequelize ORM to Raw SQL Transition
- **Initial Code Generated by AI**:
  Initial structures used Sequelize ORM models and migrations.
- **Correction**: Based on the user's feedback to make debugging easier for interviews, the AI deleted the `models` folder and rewrote the database layer to utilize raw connection pool queries using `mysql2/promise`.
